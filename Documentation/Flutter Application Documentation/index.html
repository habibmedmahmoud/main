B) Environment Setup - top

Note: You can also follow Flutter offical quick start documentation for your environment setup. Check it Now.

Running of a Flutter App is very easy and simple.These are the following steps for machine setup for an Flutter App.

If your Machine environment is already setup for running an Flutter then you can skip the steps.

  1. Mac
  2. Windows

1. Mac

  1. Download and Setup Flutter SDK
  2. Install Watchman
  3. Xcode
  4. Command Line Tools
  5. CocoaPods

A. Download and Setup Flutter SDK

git clone https://github.com/flutter/flutter.git -b stable

B. Install Watchman

brew install watchman

Watchman is a tool by Facebook for watching changes in the filesystem. It is highly recommended you install it for better performance.

C. Xcode

The easiest way to install Xcode is via the Mac App Store. Installing Xcode will also install the iOS Simulator and all the necessary tools to build your iOS app.

Download Xcode from here

D. Setting up Command Line Tools (Xcode>Preferences>Command Line Tool)

E. Install CocoaPods

sudo gem install cocoapods

    E) Firebase Push Notification - top

    1. Android
    2. iOS
    3. Flutter Code

    For Android goto the firebase console and follow below step:

    1) Android

    From the left sidebar, select the gear icon next to Project Overview and then select Project Settings:



    Scroll down to Your apps section and select the Android button:



    On the Register App section, enter the package name of your Android app.

    App nickname field is a given name for your app on Firebase console.



    Download config file will give you a Firebase configuration file. Drag the file on top of app folder under the android.


    2) iOS

    From the left sidebar, select the gear icon next to Project Overview and then select Project Settings:



    Scroll down to Your apps section and select the iOS button:



    On the Register App section, enter the bundle ID of your iOS app.



    Download config file will give you a Firebase configuration file.



    Open your iOS project in xcode and add the “Google Service-Info.list” file into your root folder(PROJECTNAME/ios) of your project from xcode by right-clicking on the project as shown below,




    Create an APN certificate on Apple developer account and mapping it on Firebase project

    In this section you need to follow the steps in the link https://firebase.flutter.dev/docs/manual-installation/ios/ to generate the APN key on the Apple developer account and then link the .p8 certificate to the firebase iOS project we have created in the above section. Make sure the below points have been achieved before proceeding further to the next section.

    Adding the capabilities “Background Modes” & “Push Notifications” for the project in xcode.



    while adding the capabilities, make sure you select “All” as shown below.




    Register a key in the Apple developer account with “Apple Push Notification Service (APNs)” enabled.

    Upload the above key file(.p8) in the firebase iOS project under APN authentication key section in firebase.

    Under the identifier, enabling the push notification service.

    If you already have the distribution certificate and developer certificate, you don’t have to worry about provisioning the certificate. Below is what you should cross verify in the xcode

    lib/network : this folder includes globally network configuration, network POST & GET common request.

    lib/provider: ProvidersFolder

    lib/ui/screens: Screems includes whole app screens

    lib/ui/widgets: includes placehoders
